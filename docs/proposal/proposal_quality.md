## 3. 成果物品質基準

### 3.1 ドキュメント品質基準

| 品質特性 | 基準 | 確認方法 |
|---------|------|---------|
| 完全性 | 仕様書要求項目の100%網羅 | チェックリスト |
| 正確性 | 技術的誤りがないこと | 専門家レビュー |
| 一貫性 | 用語・表記の統一 | 用語集との照合 |
| 可読性 | 構造化・図表の適切な使用 | レビュー |
| 追跡性 | 要件との対応付け | トレーサビリティマトリクス |

### 3.2 成果物別品質チェックリスト

```mermaid
flowchart LR
    subgraph 品質チェック
        A[形式チェック] --> B[内容チェック]
        B --> C[整合性チェック]
        C --> D[承認]
    end
```

#### 業務実施計画書

| # | チェック項目 | 確認者 |

| 2 | スケジュールが現実的か | PM |

## 4. AI精度評価

### 4.1 評価指標

```mermaid
flowchart TB
    subgraph 評価指標
        A[Accuracy<br/>正確度]
        B[Precision<br/>適合率]
        C[Recall<br/>再現率]
        D[F1 Score]
    end
    
    subgraph 算出
        E["TP + TN / Total"]
        F["TP / (TP + FP)"]
        G["TP / (TP + FN)"]
        H["2 × P × R / (P + R)"]
    end
    
    A --> E
    B --> F
    C --> G
    D --> H
```

### 4.2 評価目標値

| 指標 | 目標値 | 判定基準 |
|------|--------|---------|
| Accuracy | 80%以上 | 全チェック項目の平均 |
| Precision | 75%以上 | 全チェック項目の平均 |
| Recall | 80%以上 | 全チェック項目の平均 |
| F1 Score | 77%以上 | 全チェック項目の平均 |

### 4.3 評価プロセス

```mermaid
flowchart LR
    A[Ground Truth<br/>作成] --> B[チェック<br/>実行]
    B --> C[結果<br/>収集]
    C --> D[混同行列<br/>作成]
    D --> E[指標<br/>算出]
    E --> F{目標<br/>達成?}
    F -->|No| G[改善]
    G --> B
    F -->|Yes| H[完了]
```

### 4.4 混同行列テンプレート

| | AIが「指摘あり」 | AIが「指摘なし」 |
|---|:---:|:---:|
| **実際に問題あり** | True Positive (TP) | False Negative (FN) |
| **実際に問題なし** | False Positive (FP) | True Negative (TN) |

## 5. レビュープロセス

### 5.1 レビュー種別

```mermaid
flowchart TB
    subgraph レビュー種別
        A[セルフレビュー<br/>作成者本人]
        B[ピアレビュー<br/>チームメンバー]
        C[テクニカルレビュー<br/>TL]
        D[マネジメントレビュー<br/>PM]
        E[外部レビュー<br/>主管課]
    end
    
    A --> B --> C --> D --> E
```

### 5.2 レビュー観点

| レビュー種別 | 観点 | 実施者 |
|-------------|------|--------|
| セルフレビュー | 誤字脱字、形式、論理的整合性 | 作成者 |
| ピアレビュー | 技術的正確性、可読性 | チームメンバー |
| テクニカルレビュー | 技術選定妥当性、実現可能性 | TL |
| マネジメントレビュー | 要件充足、スケジュール整合性 | PM |
| 外部レビュー | 業務要件への適合性 | 主管課 |

### 5.3 レビュー記録

```mermaid
flowchart LR
    A[レビュー実施] --> B[指摘記録]
    B --> C[対応方針決定]
    C --> D[修正実施]
    D --> E[確認]
    E --> F[完了記録]
```

| 項目 | 内容 |
|------|------|
| レビュー日時 | YYYY/MM/DD HH:MM |
| レビュー対象 | ドキュメント名/バージョン |
| レビュアー | 氏名 |
| 指摘事項 | 内容、重要度（High/Medium/Low） |
| 対応方針 | 修正/保留/却下 |

## 7. 課題管理

### 7.1 課題管理プロセス

```mermaid
flowchart LR
    A[課題検出] --> B[登録]
    B --> C[分析]
    C --> D[対応方針決定]
    D --> E[対応実施]
    E --> F[確認]
    F --> G[クローズ]
```

### 7.2 課題重要度分類

| 重要度 | 定義 | 対応期限 |
|--------|------|---------|
| Critical | プロジェクト継続に影響 | 即日対応 |
| High | スケジュール・品質に影響 | 3日以内 |
| Medium | 改善が望ましい | 1週間以内 |
| Low | 軽微な改善 | 次フェーズで対応 |

### 7.3 課題管理表テンプレート

| ID | 登録日 | タイトル | 内容 | 重要度 | 担当 | 期限 | ステータス |
|----|--------|---------|------|--------|------|------|-----------|
| ISS-001 | YYYY/MM/DD | 課題タイトル | 詳細内容 | High | TL | YYYY/MM/DD | Open |

## 8. リスク管理

### 8.1 リスク管理プロセス

```mermaid
flowchart LR
    A[リスク特定] --> B[分析・評価]
    B --> C[対応計画策定]
    C --> D[モニタリング]
    D --> E{発生?}
    E -->|Yes| F[対応実行]
    E -->|No| D
    F --> G[評価・記録]
```

### 8.2 リスク評価マトリクス

```mermaid
quadrantChart
    title リスク評価マトリクス
    x-axis 発生確率低 --> 発生確率高
    y-axis 影響度低 --> 影響度高
    quadrant-1 高優先度対応
    quadrant-2 要監視
    quadrant-3 受容可能
    quadrant-4 軽減策検討
```

### 8.3 主要リスクと対策

| # | リスク | 発生確率 | 影響度 | 対策 |
|---|--------|:--------:|:------:|------|
| R1 | AI精度が目標未達 | 中 | 高 | 反復改善、チェック項目の絞り込み |
| R2 | スケジュール遅延 | 中 | 高 | バッファ確保、優先度付けタスク管理 |
| R3 | 要員の離脱 | 低 | 中 | ナレッジ共有、バックアップ体制 |
| R4 | 要件変更 | 中 | 中 | 変更管理プロセス、影響分析 |
| R5 | 技術的課題 | 中 | 中 | PoC段階での早期検証、代替案準備 |
| R6 | API利用コスト超過 | 低 | 低 | モニタリング、上限設定 |

### 8.4 リスク対応種別

```mermaid
flowchart TB
    subgraph リスク対応
        A[回避<br/>発生原因を除去]
        B[軽減<br/>発生確率・影響を低減]
        C[転嫁<br/>第三者へ移転]
        D[受容<br/>許容範囲として受け入れ]
    end
```
