## 2. 技術アプローチ

### 2.1 システムアーキテクチャ

**Model Context Protocol (MCP)** に基づくNeuro-Symbolicアーキテクチャ

```
┌─────────────────────────────────────────────────────────────┐
│                 MCP Host (CLI / Web UI)                      │
│                                                              │
│  ┌──────────────────────────────────────────────────────┐  │
│  │              庁内AI環境（源内）連携                   │  │
│  │              LLM Sampling 実行                        │  │
│  └──────────────────────────────────────────────────────┘  │
└──────────────────────────┬──────────────────────────────────┘
                           │ MCP Protocol (JSON-RPC 2.0)
           ┌───────────────┼───────────────┐
           ▼               ▼               ▼
┌──────────────────┐┌──────────────────┐┌──────────────────┐
│ smartreviewer-   ││ smartreviewer-   ││ smartreviewer-   │
│ core             ││ rag              ││ knowledge        │
│                  ││                  ││                  │
│ • レビュー実行   ││ • Vector検索    ││ • 知識グラフ     │
│ • 結果管理       ││ • Graph検索     ││ • オントロジー   │
│ • レポート生成   ││ • Hybrid検索    ││ • 網羅性チェック │
└──────────────────┘└──────────────────┘└──────────────────┘
           │               │               │
           ▼               ▼               ▼
┌─────────────────────────────────────────────────────────────┐
│                       Data Layer                             │
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐   │
│  │ Qdrant   │  │ Neo4j    │  │ MinIO    │  │ RDF/OWL  │   │
│  │ Vector   │  │ Graph    │  │ Storage  │  │ Ontology │   │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 ハイブリッドRAGの仕組み

```
         入力: ドキュメント + チェック項目
                      │
                      ▼
         ┌────────────────────────┐
         │   チェック種別判定     │
         └────────────────────────┘
                      │
        ┌─────────────┼─────────────┐
        ▼             ▼             ▼
┌──────────────┐┌──────────────┐┌──────────────┐
│ Vector RAG   ││ Graph RAG    ││ Hybrid RAG   │
│              ││              ││              │
│ 表記・書式   ││ 構造・整合性 ││ 複合チェック │
│ フォーマット ││ 関係性検証   ││              │
└──────────────┘└──────────────┘└──────────────┘
        │             │             │
        └─────────────┼─────────────┘
                      ▼
         ┌────────────────────────┐
         │   LLM推論（源内）      │
         │   MCP Sampling経由     │
         └────────────────────────┘
                      │
                      ▼
         出力: チェック結果 + 是正提案
```

### 2.3 MCPの利点

| 利点 | 説明 |
|------|------|
| **標準化** | AI統合の標準プロトコル、相互運用性確保 |
| **セキュリティ** | LLM認証はHost側で管理、Server分離 |
| **拡張性** | 新しいToolの追加が容易 |
| **庁内環境適合** | 源内との統合が容易 |
